📊 Barras y 📈 Líneas 4 / 18

📊 Gráfico de Barras

  • Variable cualitativa (categorías) en el eje X.
  • Magnitud (conteo, promedio) en el eje Y.
  • Barras horizontales cuando las etiquetas son largas.
Ej: N° tickets por tipo (Bug/Feature/Soporte/Seguridad)

📈 Gráfico de Líneas

  • Variable temporal (fecha/hora) en el eje X.
  • Variable continua en el eje Y.
  • Ideal para tendencias y series de tiempo.
Ej: Latencia del servidor por hora durante el mes
💡 Regla de oro
Barras para comparar entre categorías. Líneas para mostrar cambio a lo largo del tiempo.

🔵 Histograma y 📦 Boxplot 5 / 18

🔵 Histograma

  • Muestra la frecuencia de cada rango de valores.
  • Solo para variables cuantitativas continuas.
  • Los bins (intervalos) afectan la visualización — elige con cuidado.
Ej: Distribución de tiempos de respuesta HTTP (ms)

📦 Boxplot (Caja y Bigotes)

  • Muestra: Q1, Mediana, Q3, bigotes y outliers.
  • Excelente para detectar valores atípicos.
  • Permite comparar distribuciones de múltiples grupos.
Ej: Comparar tiempo de carga por servidor (detectar cuál falla)

🥧 Pastel y ⚫ Dispersión 6 / 18

🥧 Pastel/Torta

  • Muestra composición porcentual de un todo.
  • Úsalo con máximo 5 categorías.
  • Si hay muchas categorías, prefiere barras horizontales.
Ej: % de uptime por tipo de servicio (Web/DB/Cache)

⚫ Diagrama de Dispersión

  • Muestra la relación entre dos variables numéricas.
  • Permite visualizar si hay correlación.
  • Cada punto representa un registro/observación.
Ej: ¿A mayor CPU%, mayor latencia ms? → buscar correlación
⚠️ Error frecuente
El diagrama de dispersión muestra correlación, NO causalidad. Dos variables correlacionadas no significa que una cause la otra.

Python: matplotlib y seaborn
Las dos librerías estándar de Python para visualización estadística. matplotlib para control total, seaborn para gráficos elegantes con menos código.
🐍 Python: matplotlib — 4 gráficos básicos 10 / 18
import matplotlib.pyplot as plt
import numpy as np
np.random.seed(42)

fig, axes = plt.subplots(2, 2, figsize=(12, 8))

# 1. BARRAS: tickets por tipo
tipos = ['Bug','Feature','Soporte','Seguridad']
axes[0,0].bar(tipos, [145,98,234,45], color=['#ef4444','#3b82f6','#22c55e','#f59e0b'])
axes[0,0].set_title('Tickets por Tipo')

# 2. LÍNEAS: latencia por hora
horas = np.arange(24)
lat = 50 + 30*np.sin(horas*np.pi/12) + np.random.normal(0,5,24)
axes[0,1].plot(horas, lat, 'o-', color='#2563eb', linewidth=2)
axes[0,1].axhline(y=100, color='red', linestyle='--', label='SLA')
axes[0,1].set_title('Latencia por Hora (ms)')

# 3. HISTOGRAMA: tiempos de respuesta
tiempos = np.random.gamma(2, 50, 2000)
axes[1,0].hist(tiempos, bins=40, color='#8b5cf6', alpha=0.8, edgecolor='white')
axes[1,0].set_title('Distribución Tiempos de Respuesta')

# 4. DISPERSIÓN: CPU vs latencia
cpu = np.random.normal(60, 15, 200)
axes[1,1].scatter(cpu, cpu*2.5 + np.random.normal(0,30,200), alpha=0.5, color='#f59e0b')
axes[1,1].set_title('CPU% vs Latencia')
plt.tight_layout(); plt.show()

🐍 Python: seaborn — Gráficos estadísticos 11 / 18

seaborn tiene funciones específicas para análisis estadístico:

import seaborn as sns
import pandas as pd, numpy as np
np.random.seed(42)
n = 500
df = pd.DataFrame({
    'servidor': np.random.choice(['Web-1','Web-2','DB-1','Cache-1'], n),
    'cpu_pct':  np.random.beta(2, 3, n) * 100,
    'latencia': np.abs(np.random.lognormal(4, 0.5, n)),
    'estado':   np.random.choice(['OK','Warning','Critical'], n, p=[.75,.18,.07])
})
sns.set_theme(style='whitegrid')

# Boxplot de latencia por servidor
sns.boxplot(data=df, x='servidor', y='latencia', palette='Blues')
plt.title('Latencia por Servidor (Boxplot)')
plt.show()

# Histograma con densidad por estado
sns.histplot(data=df, x='cpu_pct', hue='estado', bins=25, kde=True)
plt.title('Distribución CPU% por Estado del Servidor')
plt.show()

R: ggplot2
La gramática de gráficos: datos + estética (aes) + geometría (geom). El estándar en R para visualización estadística.
📊 R: ggplot2 — Gramática de Gráficos 13 / 18

La estructura base de ggplot2: ggplot(data, aes(x,y)) + geom_*()

library(ggplot2)
library(dplyr)
set.seed(42)
n <- 500
df <- data.frame(
  servidor = sample(c("Web-1","Web-2","DB-1","Cache-1"), n, replace=TRUE),
  cpu_pct  = rbeta(n, 2, 3) * 100,
  latencia = abs(rlnorm(n, 4, 0.5)),
  estado   = sample(c("OK","Warning","Critical"), n, replace=TRUE, prob=c(.75,.18,.07))
)

# BOXPLOT por servidor
ggplot(df, aes(x=servidor, y=latencia, fill=servidor)) +
  geom_boxplot(outlier.color="red", alpha=0.8) +
  labs(title="Latencia por Servidor", x="", y="Latencia (ms)") +
  theme_minimal() + theme(legend.position="none")

# HISTOGRAMA con densidad
ggplot(df, aes(x=cpu_pct, fill=estado)) +
  geom_histogram(bins=30, alpha=0.7, position="identity") +
  scale_fill_manual(values=c(OK="seagreen",Warning="orange",Critical="red")) +
  labs(title="CPU% por Estado") + theme_minimal()

📊 R: Facetas y Series Temporales 14 / 18

facet_wrap() permite mostrar múltiples gráficos en una sola visualización:

library(ggplot2); library(dplyr)
dias <- seq(as.Date("2026-04-01"), by="day", length.out=30)
df_ts <- data.frame(
  fecha   = dias,
  usuarios = as.integer(rpois(30, 1000) + seq(0, 200, length.out=30)),
  latencia = rnorm(30, 250, 40)
)

# Gráfico de líneas con área sombreada (tendencia)
ggplot(df_ts, aes(x=fecha, y=usuarios)) +
  geom_area(fill="#3b82f6", alpha=0.3) +
  geom_line(color="#2563eb", linewidth=1.5) +
  labs(title="📈 Usuarios Activos — Tendencia Creciente",
       x="", y="Usuarios Diarios") +
  theme_minimal(base_size=12)

# Dispersión CPU vs Latencia con facetas por servidor
ggplot(df, aes(x=cpu_pct, y=latencia, color=estado)) +
  geom_point(alpha=0.5) + facet_wrap(~servidor, nrow=2) +
  scale_color_manual(values=c(OK="seagreen",Warning="orange",Critical="red")) +
  theme_minimal(base_size=9)

❌ Errores Comunes en Visualización 15 / 18

🚫 Pastel con muchas categorías

Más de 5 categorías hace el gráfico ilegible. Usa barras horizontales en su lugar.

🚫 Eje Y truncado en barras

Empezar el eje Y en 100 en lugar de 0 exagera las diferencias visuales. Siempre inicia en 0.

🚫 Histograma para categorías

El histograma es para variables continuas. Para categorías usa barras o pastel.

🚫 Correlación = Causalidad

Si CPU% y latencia se correlacionan, no significa que una cause la otra. Puede haber una variable oculta.

✅ Buenas prácticas

Siempre incluye: título descriptivo, etiquetas de ejes, unidades de medida y fuente de datos.

✅ Contexto es clave

Un gráfico sin contexto no comunica. Añade líneas de referencia (SLA, umbral crítico) para dar significado.

📊 Comparativa: matplotlib vs seaborn vs ggplot2 16 / 18

🐍 matplotlib

  • Control total del gráfico
  • Más verboso
  • Ideal: dashboards complejos
  • fig, ax = plt.subplots()

🐍 seaborn

  • Construido sobre matplotlib
  • Menos código
  • Ideal: análisis estadístico rápido
  • sns.boxplot(data=df, x, y)

📊 ggplot2 (R)

  • Gramática declarativa
  • Muy potente + facetas
  • Ideal: publicaciones científicas
  • ggplot(aes) + geom_*()
💡 Consejo profesional
En proyectos de Python, usa seaborn para exploración rápida y matplotlib para personalización final. En R, ggplot2 es el estándar indiscutible.
