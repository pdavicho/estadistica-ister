Muestreo Probabilístico
Cuando cada elemento de la población tiene una probabilidad conocida de ser seleccionado. Garantiza representatividad estadística.
Aleatorio Simple (MAS) 4 / 18
Probabilístico P = n/N
📖 Definición
Cada elemento de la población tiene la misma probabilidad de ser seleccionado. Se usa un generador de números aleatorios.
  • Ventaja: Simple de implementar y sin sesgo de selección.
  • Uso: Cuando la población es homogénea y tiene una lista completa.
  • Ejemplo: Seleccionar 200 clientes de 10,000 con np.random.choice() en Python.
💡 Nota
Con reemplazo: el mismo elemento puede ser seleccionado más de una vez. Sin reemplazo: cada elemento aparece solo una vez en la muestra.

Muestreo Sistemático 5 / 18
Probabilístico k = N/n
📖 Definición
Se elige un punto de inicio aleatorio y luego se selecciona cada k-ésimo elemento de la lista ordenada.
Ejemplo paso a paso:
N=1000 servidores, n=50. k = 1000/50 = 20. Inicio aleatorio: 7. Seleccionados: 7, 27, 47, 67, 87... ✓
  • Ventaja: Rápido y fácil de aplicar en listas ordenadas.
  • Riesgo: Si la lista tiene un patrón cíclico cada k elementos, se puede introducir sesgo.

Muestreo Estratificado 6 / 18
Probabilístico nₕ = (Nₕ/N) × n
📖 Definición
La población se divide en grupos homogéneos (estratos) y se muestrea de cada uno proporcionalmente.

📊 Cuándo usarlo

Cuando la población tiene subgrupos claramente definidos con características distintas. Ej: departamentos, regiones, géneros.

✅ Ventaja

Es el más preciso estadísticamente porque garantiza representación de todos los grupos relevantes.

💡 Ejemplo
Empresa 2000 empleados: TI(800), Ventas(250), RRHH(150)... Para n=200: TI=80, Ventas=25, RRHH=15...

Muestreo por Conglomerados 7 / 18
Probabilístico Grupos = m
📖 Definición
La población se divide en grupos heterogéneos (conglomerados) que reflejan la diversidad de la población. Se seleccionan grupos completos al azar.

🔑 Diferencia clave con Estratificado

Estratificado: grupos homogéneos, se muestrea de cada uno.
Conglomerados: grupos heterogéneos, se estudia el grupo completo.

📌 Ejemplo

Estudiar empleados de un banco: seleccionar 5 sucursales (conglomerados) al azar y encuestar a todos los empleados de esas 5 sucursales.

Muestreo No Probabilístico
Cuando la selección no garantiza probabilidades iguales. Más rápido pero con riesgo de sesgo.
Técnicas No Probabilísticas 10 / 18
⚠️ Atención
No todos los elementos tienen la misma probabilidad. Son más rápidos y económicos, pero no permiten inferencia estadística formal. Se usan en estudios exploratorios.
No Prob.

⚡ Conveniencia

Elementos más accesibles. Rápido pero con alto sesgo. Ej: encuestar solo usuarios en línea ahora mismo.

No Prob.

📋 Por Cuotas

Cuotas por características (edad, género) pero selección discrecional dentro. Ej: entrevistar 50H y 50M.

No Prob.

❄️ Bola de Nieve

Participantes reclutan a nuevos. Para poblaciones difíciles. Ej: freelancers tech — cada uno recomienda a otro.

Tipos de Variables
Antes de analizar datos, debemos identificar qué tipo de variable estamos midiendo. Esto determina qué métodos estadísticos podemos aplicar.
2.4 Variables Cualitativas y Cuantitativas 12 / 18

🏷 Cualitativas (Categóricas)

Expresan cualidades o atributos. No se operan matemáticamente.

Nominales: Sin orden — SO: Windows/Linux, color, tipo de error
Ordinales: Con orden — Prioridad: Crítico>Alto>Bajo, rating

🔢 Cuantitativas (Numéricas)

Expresan cantidades. Permiten operaciones matemáticas (suma, promedio, etc.).

Discretas: Enteros contables — N° errores, N° commits, N° clientes
Continuas: Cualquier valor — Latencia ms, temperatura CPU%, tiempo hrs

2.5 Escalas de Medición (NOIR) 13 / 18

Las escalas determinan qué operaciones estadísticas podemos aplicar:

EscalaCaracterísticasOperacionesEj. en Sistemas
🏷 NominalSolo categorías, sin ordenModa, frecuenciasTipo SO: Win/Linux
📶 OrdinalOrden, sin distancia definidaModa, MedianaPrioridad: Crit>Alto>Bajo
🌡 IntervaloOrden + distancias. Cero arbitrarioMedia, Desv. std.Temperatura °C del servidor
⚖️ RazónOrden + distancias. Cero absolutoTodas + cocientesLatencia (ms), uptime (%)
💡 Regla
Nominal → Ordinal → Intervalo → Razón (más completa). Solo la escala de Razón tiene un cero verdadero (ausencia real).

🐍 Python: Muestreo Estratificado 14 / 18

Muestreo estratificado proporcional por departamento:

import pandas as pd, numpy as np
np.random.seed(42)
deptos = {'TI':800,'Ventas':250,'RRHH':150,'Finanzas':200,'Marketing':100}
N = 2000; n = 200
filas = []
for d, c in deptos.items():
    for sal in np.random.normal(800, 150, c):
        filas.append({'depto':d,'salario':round(sal,2)})
df = pd.DataFrame(filas)

# Muestreo estratificado proporcional
muestra = df.groupby('depto', group_keys=False).apply(
    lambda x: x.sample(int(round(len(x)/N*n)), random_state=42))
print(muestra.groupby('depto').size())
print(f"Media real: ${df.salario.mean():.0f} | Muestra: ${muestra.salario.mean():.0f}")
TI=80, Ventas=25, RRHH=15, Finanzas=20, Marketing=10 → total=150
Media real: $800 | Muestra: $801

📊 R: Muestreo Estratificado (dplyr) 15 / 18
library(dplyr)
set.seed(42)
df <- data.frame(
  depto = rep(c("TI","Ventas","RRHH","Finanzas","Marketing"),
              c(800,250,150,200,100)),
  salario = rnorm(1500, 800, 150)
)

# Muestreo estratificado proporcional (10%)
muestra <- df |>
  group_by(depto) |>
  slice_sample(prop = 0.1) |>
  ungroup()

cat(sprintf("Población: %d | Muestra: %d\n", nrow(df), nrow(muestra)))
cat(sprintf("Media real: $%.0f | Muestra: $%.0f\n",
            mean(df$salario), mean(muestra$salario)))
count(muestra, depto)

Media real: $800 | Muestra: $801

⚠️ Errores Comunes en Muestreo 16 / 18

❌ Error de Sesgo

Cuando la muestra no representa correctamente a la población. Causa: selección no aleatoria o cuestionario mal diseñado.

❌ Error de Muestreo

Diferencia entre el estadístico de la muestra y el parámetro real. Siempre existe, pero se puede minimizar aumentando n.

❌ Muestra Pequeña

Si n es muy pequeño, los resultados son poco confiables. Regla general: n ≥ 30 para distribuciones simétricas.

✅ Cómo evitarlos

Aleatorización correcta + tamaño de muestra adecuado + estratificación cuando hay subgrupos relevantes.
