¿Qué es una Distribución de Frecuencias? 2 / 18
📖 Definición
Una distribución de frecuencias es una tabla que organiza un conjunto de datos mostrando, para cada valor o categoría, cuántas veces aparece (su frecuencia). Es la forma más básica de resumir datos.

Bienvenido al Tema 2. Ya sabes identificar variables (S2), graficarlas (S3) y explorarlas con EDA (S4). Ahora aprenderás a resumirlas en tablas — el paso previo a cualquier gráfico de distribución.

🎯 ¿Para qué sirve?

Convierte cientos o miles de datos individuales en un resumen comprensible de un vistazo.

🔗 Conexión

Toda tabla de frecuencias es la base de un histograma, un gráfico de barras o un polígono de frecuencias.

💡 Analogía
Es como pasar lista: en lugar de leer 40 nombres uno por uno, cuentas "30 presentes, 10 ausentes" — ya tienes una distribución de frecuencias de 2 categorías.

¿Por qué Construir Tablas de Frecuencias? 3 / 18

📊 Resumir

Reduce un conjunto de datos extenso a una tabla compacta y fácil de leer.

⚖️ Comparar

Permite comparar categorías o intervalos entre sí: ¿cuál es más común? ¿cuál es rara?

🔍 Detectar patrones

Revela qué valores concentran los datos y cuáles son atípicos, antes de graficar.

🧩 El símbolo n
En toda tabla de frecuencias, n representa el número total de datos (el tamaño de la muestra). La suma de todas las frecuencias absolutas siempre es igual a n.
⚠️ Antes de construir la tabla
Recuerda identificar primero el tipo de variable (Semana 2): si es cualitativa o cuantitativa discreta, agrupas por categorías; si es cuantitativa continua, agrupas por intervalos de clase (lo veremos en la Semana 7).

Los 4 Tipos de Frecuencia
Absoluta, relativa, porcentual y acumulada — cada una responde una pregunta distinta sobre tus datos.
Frecuencia Absoluta (fᵢ) 5 / 18
📖 Definición
La frecuencia absoluta (fᵢ) es el número de veces que se repite un valor o categoría en el conjunto de datos. Responde: "¿cuántos hay?"
Σ fᵢ = n
La suma de todas las frecuencias absolutas es igual al total de datos (n)

📋 Ejemplo: Medio de transporte de 40 estudiantes

Bus → fᵢ = 16  |  Auto propio → fᵢ = 8  |  Caminando → fᵢ = 9  |  Bicicleta → fᵢ = 4  |  Taxi/App → fᵢ = 3

Comprobación: 16+8+9+4+3 = 40 = n ✓

Frecuencia Relativa (hᵢ) y Porcentual 6 / 18
📖 Definición
La frecuencia relativa (hᵢ) es la proporción que representa cada valor respecto al total. Responde: "¿qué parte del total es?" Multiplicada por 100 se obtiene la frecuencia porcentual.
hᵢ = fᵢ / n   ·   %ᵢ = hᵢ × 100
La frecuencia relativa siempre está entre 0 y 1; la suma de todas las hᵢ es igual a 1 (o 100%)

📋 Ejemplo: continuando con los 40 estudiantes

Bus → hᵢ = 16/40 = 0.40 → 40%  |  Auto propio → 8/40 = 0.20 → 20%  |  Caminando → 9/40 = 0.225 → 22.5%

💡 ¿Por qué usar frecuencia relativa?
Permite comparar grupos de distinto tamaño. "16 estudiantes usan bus" no se compara directamente con "60 de otra sección", pero "40%" sí.

Frecuencia Acumulada (Fᵢ) y Relativa Acumulada (Hᵢ) 7 / 18
📖 Definición
La frecuencia acumulada (Fᵢ) suma progresivamente las frecuencias absolutas hasta cada categoría. Responde: "¿cuántos datos hay hasta este punto o por debajo de él?" Su versión relativa (Hᵢ) se expresa como proporción o porcentaje.
Fᵢ = f₁ + f₂ + ... + fᵢ   ·   Hᵢ = Fᵢ / n
La última frecuencia acumulada siempre es igual a n (y Hᵢ final = 1, o 100%)

📋 Ejemplo: orden Bus, Auto, Caminando, Bicicleta, Taxi

Fᵢ: 16 → 24 → 33 → 37 → 40

Lectura: "33 de los 40 estudiantes (82.5%) se movilizan en Bus, Auto propio o Caminando".

⚠️ Solo tiene sentido lógico cuando hay un orden
La frecuencia acumulada solo se interpreta de forma natural en variables cuantitativas o cualitativas ordinales. En variables nominales (Bus, Auto, Bicicleta...) el orden es arbitrario, así que se usa más con fines de cálculo que de interpretación directa.

Python: Tablas de Frecuencia con pandas
value_counts() calcula frecuencias absolutas y relativas en una sola línea — y se combina con cumsum() para acumular.
🐍 Python: value_counts() + tabla completa 11 / 18
import pandas as pd

# Encuesta: medio de transporte de 40 estudiantes ISTER
datos = ['Bus']*16 + ['Caminando']*9 + ['Auto propio']*8 + ['Bicicleta']*4 + ['Taxi/App']*3
serie = pd.Series(datos)

# Frecuencia absoluta (fi)
fi = serie.value_counts()

# Frecuencia relativa (hi) y porcentual (%i)
hi = serie.value_counts(normalize=True)

# Tabla de frecuencias completa
tabla = pd.DataFrame({
    'fi': fi,
    'hi': hi.round(3),
    '%i': (hi * 100).round(1)
})
tabla['Fi'] = tabla['fi'].cumsum()          # frecuencia acumulada
tabla['Hi%'] = tabla['%i'].cumsum().round(1) # rel. acumulada %

print(tabla)
print(f"\nVerificación: Σfi = {tabla['fi'].sum()}  (debe ser igual a n = {len(serie)})")

🐍 Python: tablas cruzadas con crosstab() 12 / 18
import pandas as pd
import numpy as np
np.random.seed(42)

# ¿El medio de transporte cambia según la jornada (Matutina/Nocturna)?
n = 120
df = pd.DataFrame({
    'transporte': np.random.choice(['Bus','Caminando','Auto propio','Bicicleta','Taxi/App'],
                                    n, p=[.40,.225,.20,.10,.075]),
    'jornada':    np.random.choice(['Matutina','Nocturna'], n, p=[.6,.4])
})

# Tabla cruzada: frecuencias absolutas
print("Frecuencia absoluta (conteo):")
print(pd.crosstab(df['transporte'], df['jornada'], margins=True, margins_name='Total'))

# Tabla cruzada: frecuencia relativa por fila (normalize='index')
print("\nFrecuencia relativa por jornada (% dentro de cada columna):")
print((pd.crosstab(df['transporte'], df['jornada'], normalize='columns') * 100).round(1))

R: Tablas de Frecuencia con table()
table() y prop.table() son el equivalente directo de value_counts() en R.
📊 R: table(), prop.table() y cumsum() 14 / 18
datos <- c(rep("Bus", 16), rep("Caminando", 9), rep("Auto propio", 8),
           rep("Bicicleta", 4), rep("Taxi/App", 3))

# Frecuencia absoluta (fi)
fi <- table(datos)
fi <- sort(fi, decreasing = TRUE)

# Frecuencia relativa (hi) y porcentual
hi  <- prop.table(fi)
pct <- round(hi * 100, 1)

# Tabla completa con acumuladas
tabla <- data.frame(
  categoria = names(fi),
  fi  = as.vector(fi),
  hi  = round(as.vector(hi), 3),
  pct = as.vector(pct),
  Fi  = cumsum(as.vector(fi)),
  Hi_pct = round(cumsum(as.vector(pct)), 1)
)
print(tabla)
cat(sprintf("\nVerificación: suma(fi) = %d  (n = %d)\n", sum(tabla$fi), length(datos)))

❌ Errores Comunes al Construir Tablas de Frecuencia 15 / 18
❌
Error 1: Que Σfᵢ no sea igual a n
Si las frecuencias absolutas no suman el total de datos, alguna categoría quedó mal contada o duplicada. Siempre verifica esta suma primero.
❌
Error 2: Confundir frecuencia relativa con porcentaje
hᵢ = 0.40 y %ᵢ = 40% son la misma proporción expresada de dos formas. Reportar "0.40%" en lugar de "40%" cambia completamente el significado.
❌
Error 3: Acumular frecuencias en variables nominales sin orden lógico
"Hasta Bicicleta llevamos el 92.5%" no tiene sentido si el orden de las categorías (Bus, Auto, Caminando...) es arbitrario. La acumulación es natural en datos ordinales o cuantitativos.
❌
Error 4: Redondear demasiado pronto
Redondear cada hᵢ antes de sumarlas puede hacer que Σhᵢ ≠ 1 (ej. 0.40+0.23+0.20+0.10+0.08 = 1.01). Calcula con precisión completa y redondea solo al presentar la tabla final.

¿Cuándo Usar Cada Tipo de Frecuencia? 16 / 18

📊 Frecuencia absoluta (fᵢ)

  • Cuando el número real de casos importa para la decisión (ej. "se necesitan 16 buses").
  • Para verificar la consistencia de los datos (Σfᵢ = n).

📐 Frecuencia relativa / porcentual (hᵢ, %ᵢ)

  • Para comparar grupos de tamaños distintos (40% en Sierra vs 60% en Costa).
  • En reportes y presentaciones — los porcentajes son más intuitivos para el público.

📈 Frecuencia acumulada (Fᵢ, Hᵢ)

  • Para responder "¿cuántos están por debajo / hasta cierto valor?" (ej. percentiles, medianas).
  • Es la base para construir la ojiva (gráfico de frecuencias acumuladas, Semana 8).
🔑 En la práctica
Una buena tabla de frecuencias casi nunca usa un solo tipo: combina fᵢ + %ᵢ + Fᵢ para que cualquier lector encuentre la lectura que necesita.
