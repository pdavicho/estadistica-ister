¿Qué es el EDA? 2 / 18
📖 Definición
El EDA (Exploratory Data Analysis) es el proceso sistemático de explorar un dataset antes de aplicar modelos estadísticos. Permite entender la estructura, distribución y calidad de los datos.

El EDA integra todo lo aprendido en el Tema 1:

📊 S1 + S2

Estadística, tipos de variables y técnicas de muestreo — ¿qué datos tengo y cómo los obtuve?

📈 S3

Representación gráfica — ¿cómo visualizo las distribuciones y relaciones?

💡 Analogía
El EDA es como hacer un "chequeo médico completo" a los datos antes de operarlos.

🎯 Objetivo del EDA
Antes de tomar decisiones estadísticas, necesitas conocer tus datos — ¿qué contienen? ¿son confiables? ¿hay anomalías?
⚠️ Error frecuente
Saltarse el EDA y aplicar directamente un modelo estadístico puede llevar a conclusiones completamente incorrectas con datos sucios o mal distribuidos.

Paso 1: Inspección · Paso 2: Calidad 4 / 18

🔍 Paso 1: Inspección

  • ¿Cuántas filas y columnas? (df.shape)
  • ¿Qué tipos de datos? (df.dtypes / str(df))
  • ¿Cómo se ven los primeros registros? (df.head())
  • ¿Cuáles son los nombres de las variables?

⚠️ Paso 2: Calidad

  • ¿Hay valores nulos? (df.isnull().sum())
  • ¿Hay duplicados? (df.duplicated().sum())
  • ¿Los tipos son correctos? (fechas como string, números como object…)
  • ¿Hay valores imposibles? (edad=-5, precio=-100)
💡 Regla
Los datos nunca están limpios la primera vez. El 80% del tiempo en un proyecto de datos se gasta limpiando y entendiendo los datos.

Media vs Mediana: El Sesgo de los Datos 6 / 18
$2,500
Media salarial
Distorsionada por salarios de CEO y directivos muy altos.
$1,800
Mediana salarial
El valor que divide en 50/50 — más representativo del "empleado típico".
📐 Interpretación del sesgo:
• Media > Mediana → Sesgo positivo (cola hacia valores altos). Ej: salarios, precios de vivienda.
• Media < Mediana → Sesgo negativo (cola hacia valores bajos).
• Media ≈ Mediana → Distribución aproximadamente simétrica.
⚠️ Cuándo usar cada una
Con outliers o sesgo: usa la Mediana. Con distribución simétrica sin outliers: la Media es igual de válida.

📈 Paso 4: Visualización

  • Histograma para ver la distribución de cada variable cuantitativa.
  • Boxplot para detectar outliers y comparar grupos.
  • Dispersión para explorar relaciones entre variables.
  • Barras para comparar frecuencias de variables categóricas.

📝 Paso 5: Conclusiones

  • ¿Qué distribución tienen los datos?
  • ¿Hay variables que se correlacionan?
  • ¿Cuáles son los outliers y qué significan?
  • ¿Los datos son suficientes y confiables para inferir?

Python: EDA con pandas
pandas.describe() genera los estadísticos básicos de un DataFrame en una sola línea. Es el punto de partida de todo EDA en Python.
🐍 Python: df.describe() + inspección 9 / 18
import pandas as pd
import numpy as np
np.random.seed(42)

# Dataset e-commerce Ecuador
df = pd.DataFrame({
    'precio':   np.abs(np.random.lognormal(3.5, 0.8, 1000)).round(2),
    'cantidad': np.random.randint(1, 6, 1000),
    'rating':   np.random.choice([1,2,3,4,5], 1000, p=[.05,.10,.20,.40,.25]),
    'producto': np.random.choice(['Electrónico','Ropa','Alimentos'], 1000),
    'region':   np.random.choice(['Sierra','Costa','Oriente'], 1000)
})
df['total'] = (df['precio'] * df['cantidad']).round(2)

# PASO 1: Inspección
print(f"Shape: {df.shape}")   # (1000, 6)
print(df.dtypes)

# PASO 2: Calidad
print(f"Nulos: {df.isnull().sum().sum()}")

# PASO 3: Estadísticos (1 línea!)
print(df.describe().round(2))

🐍 Python: EDA Visual completo 10 / 18
import matplotlib.pyplot as plt

fig, axes = plt.subplots(2, 2, figsize=(12, 8))
fig.suptitle('EDA: E-Commerce Ecuador', fontweight='bold')

# Histograma de precios (con media y mediana)
axes[0,0].hist(df['precio'], bins=35, color='#3b82f6', alpha=0.8, edgecolor='white')
axes[0,0].axvline(df.precio.mean(), color='red', linestyle='--', label=f"Media ${df.precio.mean():.0f}")
axes[0,0].axvline(df.precio.median(), color='green', linestyle='--', label=f"Mediana ${df.precio.median():.0f}")
axes[0,0].legend(); axes[0,0].set_title('Distribución Precios')

# Boxplot por categoría (detectar outliers)
df.boxplot(column='total', by='producto', ax=axes[0,1],
           medianprops={'color':'red','linewidth':2})
axes[0,1].set_title('Total por Categoría')

# Ventas por región
df.groupby('region')['total'].sum().plot(kind='bar', ax=axes[1,0], color='#8b5cf6')
axes[1,0].set_title('Ventas por Región')

# Distribución de ratings
df.rating.value_counts().sort_index().plot(kind='bar', ax=axes[1,1], color='#f59e0b')
axes[1,1].set_title(f'Ratings (Promedio: {df.rating.mean():.1f}/5)')

plt.tight_layout(); plt.show()

library(ggplot2); library(dplyr)
set.seed(42)

df <- data.frame(
  precio   = abs(rlnorm(1000, 3.5, 0.8)),
  cantidad = sample(1:5, 1000, replace=TRUE),
  rating   = sample(1:5, 1000, replace=TRUE, prob=c(.05,.10,.20,.40,.25)),
  producto = sample(c("Electrónico","Ropa","Alimentos"), 1000, replace=TRUE),
  region   = sample(c("Sierra","Costa","Oriente"), 1000, replace=TRUE)

# PASO 1+2: Inspección y calidad
cat(sprintf("Shape: %d filas × %d columnas\n", nrow(df), ncol(df)))

str(df)  # tipos de cada columna

❌ Errores Comunes — Parte 1 12 / 18
❌
Error 1: Usar la media cuando hay outliers extremos
Un solo salario de $100,000 en una empresa de 99 empleados con $800 promedio lleva la media a $1,790 — representando a nadie. Usa la mediana en distribuciones sesgadas.
❌
Error 2: Gráfico de barras con eje Y que no empieza en 0
Truncar el eje Y hace que diferencias pequeñas parezcan enormes visualmente. Es una forma de manipulación visual. Siempre desde 0 o advierte explícitamente.
❌
Error 3: Rellenar valores nulos con 0 sin análisis previo
Un precio nulo puede significar "producto gratuito", "datos faltantes" o "error de registro". Analiza el patrón antes de decidir cómo manejarlo.
Estadística · ISTERSemana 4 · Errores 1-311 may 2026
❌ Errores Comunes — Parte 2 13 / 18
❌
Error 4: Confundir correlación con causalidad
El precio del helado y las muertes por ahogamiento están correlacionados (verano). Ninguno causa al otro. La correlación nunca implica causalidad por sí sola.
❌
Error 5: Porcentajes sin N total
"El 50% tuvo problemas." ¿De 4 personas o de 40,000? El contexto del N es crucial para la interpretación estadística. Siempre indica el tamaño de la muestra.
✅ La regla general
Cualquier conclusión estadística debe incluir: el estadístico, el tamaño de la muestra, el nivel de confianza y las limitaciones del análisis.
