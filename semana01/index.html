📌 1.1 ¿Qué es la Estadística?

📖 Definición Técnica
Es la ciencia de recolectar, organizar y analizar datos para la toma de decisiones informadas.

En el contexto de Sistemas y Gestión de Data, la estadística es la herramienta que transforma el "ruido" de los datos masivos en "señal" de conocimiento útil.

  • Permite identificar patrones en grandes volúmenes de datos.
  • Facilita la toma de decisiones basada en evidencia científica, no en intuición.
  • Es el fundamento de Machine Learning, Business Intelligence y Data Science.
💡 Analogía clave
Sin estadística, un data center lleno de registros es solo ruido. Con estadística, esos mismos datos se convierten en decisiones estratégicas.

📌 1.2 Importancia en la Gestión de Data

🎯
Toma de Decisiones
Brinda información fundamentada para decisiones estratégicas.
🏆
Calidad Empresarial
Mejora procesos mediante el análisis de evidencia científica.
🔭
Perspectiva Holística
Visión integral para implementar soluciones responsables.

📌 1.3 División de la Estadística

📊 Estadística Descriptiva

"Describe lo que ya ocurrió — ¿Qué pasó?"

Organiza y resume datos mediante gráficos y tablas.

Descriptiva (D)

Ej: Reporte de ventas diarias o dashboard de tráfico de red.

🔮 Estadística Inferencial

"Predice lo que podría ocurrir — ¿Qué esperamos?"

Realiza deducciones sobre una población basándose en una muestra.

Inferencial (I)

Ej: Predecir el éxito de un software encuestando solo a un grupo.

🎯 Ejercicio: ¿D o I?

💻 Ejercicio 1

📌 1.4 Población y Muestra

🌍 Población (N)

Puede ser finita o infinita. Conjunto total de elementos que son objeto de estudio.

Ej: Base de datos completa de clientes de un banco (1 millón de registros).

🎯 Muestra (n)

Subconjunto representativo extraído de la población para su análisis.

Ej: 5000 clientes seleccionados aleatoriamente para una encuesta.

⚠️ Principio fundamental
La selección debe ser ética y científica para evitar sesgos en los resultados. ¿Cómo garantizamos que el 1% de los datos represente a todos?

🐍 1.5 Python: Simulación de Muestreo

Implementamos la diferencia entre media poblacional y muestral, demostrando por qué las muestras bien seleccionadas son representativas.

import numpy as np
np.random.seed(42)

# Población de 10,000 salarios (media: $1000, SD: $500)
poblacion = np.random.normal(1000, 500, 10000)

# Muestra aleatoria de 100 empleados (solo 1% de la población)
muestra = np.random.choice(poblacion, 100, replace=False)

print(f"Media Real (Población):  ${poblacion.mean():.2f}")
print(f"Media Estimada (Muestra): ${muestra.mean():.2f}")
print(f"Diferencia:              ${abs(poblacion.mean()-muestra.mean()):.2f}")
print(f"Representatividad:       {len(muestra)/len(poblacion)*100:.1f}%")
import numpy as np

np.random.seed(42)
poblacion = np.random.normal(1000, 500, 10000)

# Simulamos 500 muestras distintas de tamaño 100
medias = [np.random.choice(poblacion, 100, replace=False).mean() for _ in range(500)]
medias = np.array(medias)

print("=== TEOREMA DEL LÍMITE CENTRAL ===")
print(f"Media poblacional:              ${poblacion.mean():.2f}")
print(f"Promedio de 500 medias muest.:  ${medias.mean():.2f}")
print(f"Error estándar observado:       ${medias.std():.2f}")
print(f"\n→ Las medias muestrales convergen a la media real.")
Output
Media Real (Población): $999.87 | Media Estimada: $1004.52 | Diferencia: $4.65 | Representatividad: 1.0%
💡 ¿Qué observamos?
Con solo el 1% de la población (100 de 10,000), la muestra estima la media real con menos de $5 de diferencia. Esto es el poder de una muestra representativa.

📊 1.6 R: Simulación de Muestreo

set.seed(42)

# Población de 10,000 salarios
poblacion <- rnorm(10000, mean = 1000, sd = 500)

# Muestra aleatoria de 100 empleados
muestra <- sample(poblacion, size = 100, replace = FALSE)

cat("=== ESTADÍSTICAS DE MUESTREO ===\n")
cat(sprintf("Media Real (Población):   $%.2f\n", mean(poblacion)))
cat(sprintf("Media Estimada (Muestra): $%.2f\n", mean(muestra)))
cat(sprintf("Diferencia:               $%.2f\n", abs(mean(poblacion)-mean(muestra))))
cat(sprintf("Representatividad:        %.1f%%\n", length(muestra)/length(poblacion)*100))
library(ggplot2)
set.seed(42)

poblacion <- rnorm(10000, mean = 1000, sd = 500)

# 500 muestras → Teorema del Límite Central
medias <- replicate(500, mean(sample(poblacion, 100)))

df <- data.frame(media = medias)

ggplot(df, aes(x = media)) +
  geom_histogram(bins = 30, fill = "steelblue", alpha = 0.8, color = "white") +
  geom_vline(xintercept = mean(poblacion), color = "red", linetype = "dashed", linewidth = 1) +
  labs(title = "Distribución de 500 Medias Muestrales (TLC)",

       x = "Media muestral ($)", y = "Frecuencia") +

⚖️ 1.7 Ética y Valores del Analista de Datos

🎯 Honestidad y Ética
Vitales para evitar sesgos y desinformación. Un dato manipulado puede destruir la confianza en una organización.
🧠 Pensamiento Crítico
Analizar causas y soluciones considerando implicaciones éticas. No aceptar datos sin cuestionarlos.
📊 Transparencia
Reportar los métodos utilizados, el tamaño de la muestra y las limitaciones del análisis.
🔒 Privacidad
Respetar la privacidad de los individuos en los datos que analizamos.

Caso 1: Control de Calidad en Data Center

ABP · 20 min
Escenario: Una empresa recibe un lote de 5,000 discos duros. Se quiere verificar si cumplen con la vida útil prometida (≥ 3 años), pero probar los 5,000 es inviable económicamente. Se decide probar una muestra de 50 unidades seleccionadas aleatoriamente.
  1. 1
    ¿Cuál es la Población (N) en este problema?
    N = 5,000 discos duros del lote completo.
  2. 2
    ¿Cuál es la Muestra (n)?
    n = 50 discos seleccionados aleatoriamente para la prueba.
  3. 3
    ¿Es descriptivo o inferencial si proyectamos el resultado al lote completo?
    Inferencial — usamos 50 unidades para predecir el comportamiento de los 5,000.
  4. 4
    ¿Qué riesgos existen si los 50 discos NO se seleccionan aleatoriamente (ej: solo los del primer palet)?
    Sesgo de selección: los resultados no representan al total del lote. La calidad del primer palet puede diferir del resto.
  5. 5
    Si 3 de los 50 discos fallan, ¿cuántos fallos estimarías en los 5,000? Muestra tu cálculo.
    Tasa de fallo = 3/50 = 6%. Estimación = 6% × 5,000 = 300 discos podrían fallar en el lote.

Caso 2: Análisis de Tráfico Web

ABP · 15 min
Escenario: Un sitio de e-commerce recibió 1 millón de visitas ayer. El software de analítica muestra que el promedio de permanencia fue de 2.5 minutos y el 12% realizó una compra.
  1. 1
    ¿El dato "promedio de permanencia = 2.5 minutos" es descriptivo o inferencial?
    Descriptivo — describe exactamente lo que ocurrió ayer con el 100% de las visitas registradas.
  2. 2
    Propón un uso inferencial de estos mismos datos.
    Estimar que la próxima semana también tendrá ~2.5 min de permanencia y ~12% de conversión, usando el día de ayer como muestra representativa del comportamiento futuro.
  3. 3
    ¿La población aquí es finita o infinita?
    Finita: 1 millón de visitas de ese día. Infinita: si consideramos todas las visitas futuras posibles.
  4. 4
    El jefe quiere proyectar que "este mes habrá 30 millones de visitas". ¿Es válida esta inferencia? ¿Qué factores podrían sesgarla?
    Parcialmente válida. Sesgos posibles: el día analizado puede ser inusual (promoción, campaña), hay variaciones por día de semana y factores externos (feriados, competencia).
