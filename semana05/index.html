📌 5.1 ¿Qué es una Distribución de Frecuencias?

📖 Definición
Una distribución de frecuencias es una tabla que organiza un conjunto de datos mostrando, para cada valor o categoría, cuántas veces aparece (su frecuencia). Es la forma más básica — y más útil — de resumir datos.

Iniciamos el Tema 2. Ya sabes identificar variables y escalas (Semana 2), graficarlas (Semana 3) y explorarlas con EDA (Semana 4). Ahora aprenderás a resumirlas en tablas de frecuencia — el paso previo a cualquier histograma, gráfico de barras o polígono de frecuencias.

📊
Resumir
Convierte cientos de datos individuales en una tabla comprensible de un vistazo.
⚖️
Comparar
Permite comparar categorías o intervalos: ¿cuál es más común? ¿cuál es rara?
🔍
Detectar patrones
Revela qué valores concentran los datos antes de graficar o modelar.
💡 Analogía
Es como pasar lista: en lugar de leer 40 nombres uno por uno, dices "30 presentes, 10 ausentes" — ya construiste una distribución de frecuencias de 2 categorías.
🧩 El símbolo n
En toda tabla de frecuencias, n representa el número total de datos (tamaño de la muestra). La suma de todas las frecuencias absolutas siempre es igual a n. Antes de construir la tabla, identifica el tipo de variable (Semana 2): si es cualitativa o cuantitativa discreta con pocos valores, agrupas por categorías; si es cuantitativa continua, agrupas por intervalos de clase (lo veremos en la Semana 7).

📌 5.2 Los 4 Tipos de Frecuencia

TipoSímboloFórmula¿Qué responde?
AbsolutafᵢConteo directo¿Cuántos hay en esta categoría?
Relativahᵢfᵢ / n¿Qué proporción del total representa?
Porcentual%ᵢhᵢ × 100¿Qué porcentaje del total representa?
AcumuladaFᵢf₁+f₂+...+fᵢ¿Cuántos hay hasta esta categoría (inclusive)?
Rel. AcumuladaHᵢFᵢ / n¿Qué proporción acumula hasta este punto?
📖 Frecuencia absoluta y relativa — ejemplo
Encuesta a 40 estudiantes sobre su medio de transporte: Bus → fᵢ=16 → hᵢ=16/40=0.40 (40%); Auto propio → fᵢ=8 → hᵢ=0.20 (20%); Caminando → fᵢ=9 → hᵢ=0.225 (22.5%). Comprobación: 16+8+9+4+3 = 40 = n ✓
📋 Frecuencia acumulada — ejemplo
Ordenando de mayor a menor frecuencia (Bus, Caminando, Auto propio, Bicicleta, Taxi/App), la Fᵢ avanza así: 16 → 25 → 33 → 37 → 40. Lectura: "33 de los 40 estudiantes (82.5%) se movilizan en Bus, Caminando o Auto propio".
💡 Regla de oro
Una tabla de frecuencias completa casi siempre incluye fᵢ, hᵢ (o %ᵢ) y Fᵢ juntas — así cualquier lector encuentra la lectura que necesita sin recalcular nada. Y siempre verifica: Σfᵢ = n y Σhᵢ = 1 (o 100%).

📌 5.3 Construcción de una Tabla de Frecuencias

1
Listar categorías
Identifica los valores únicos de la variable
2
Contar (fᵢ)
Cuenta cuántas veces aparece cada valor
3
Calcular hᵢ y %ᵢ
Divide cada fᵢ entre n y multiplica por 100
4
Acumular (Fᵢ, Hᵢ)
Suma progresivamente fᵢ y %ᵢ
5
Verificar
Σfᵢ=n · Σhᵢ=1 · última Fᵢ=n

⚠️ Solo tiene sentido lógico cuando hay un orden
La frecuencia acumulada se interpreta de forma natural en variables cuantitativas o cualitativas ordinales. En variables nominales (Bus, Auto, Bicicleta...) el orden es arbitrario, así que se usa más con fines de cálculo que de interpretación directa.

🐍 5.4 Python: Tablas de Frecuencia con pandas

import pandas as pd

# Encuesta: medio de transporte de 40 estudiantes ISTER
datos = ['Bus']*16 + ['Caminando']*9 + ['Auto propio']*8 + ['Bicicleta']*4 + ['Taxi/App']*3
serie = pd.Series(datos)

# Frecuencia absoluta (fi)
fi = serie.value_counts()

# Frecuencia relativa (hi) y porcentual (%i)
hi = serie.value_counts(normalize=True)

# Tabla de frecuencias completa
tabla = pd.DataFrame({
    'fi': fi,
    'hi': hi.round(3),
    '%i': (hi * 100).round(1)
})
tabla['Fi']  = tabla['fi'].cumsum()           # frecuencia acumulada
tabla['Hi%'] = tabla['%i'].cumsum().round(1)  # rel. acumulada %

print(tabla)
print(f"\nVerificación: Σfi = {tabla['fi'].sum()}  (debe ser igual a n = {len(serie)})")
import pandas as pd
import numpy as np
np.random.seed(42)

# ¿El medio de transporte cambia según la jornada (Matutina/Nocturna)?
n = 120
df = pd.DataFrame({
    'transporte': np.random.choice(['Bus','Caminando','Auto propio','Bicicleta','Taxi/App'],
                                    n, p=[.40,.225,.20,.10,.075]),
    'jornada':    np.random.choice(['Matutina','Nocturna'], n, p=[.6,.4])
})

# Tabla cruzada: frecuencias absolutas
print("Frecuencia absoluta (conteo):")
print(pd.crosstab(df['transporte'], df['jornada'], margins=True, margins_name='Total'))

# Tabla cruzada: frecuencia relativa por columna (normalize='columns')
print("\nFrecuencia relativa por jornada (% dentro de cada columna):")
print((pd.crosstab(df['transporte'], df['jornada'], normalize='columns') * 100).round(1))

📊 5.5 R: Tablas de Frecuencia con table() y prop.table()

datos <- c(rep("Bus", 16), rep("Caminando", 9), rep("Auto propio", 8),
           rep("Bicicleta", 4), rep("Taxi/App", 3))

# Frecuencia absoluta (fi)
fi <- table(datos)
fi <- sort(fi, decreasing = TRUE)

# Frecuencia relativa (hi) y porcentual
hi  <- prop.table(fi)
pct <- round(hi * 100, 1)

# Tabla completa con acumuladas
tabla <- data.frame(
  categoria = names(fi),
  fi  = as.vector(fi),
  hi  = round(as.vector(hi), 3),
  pct = as.vector(pct),
  Fi  = cumsum(as.vector(fi)),
  Hi_pct = round(cumsum(as.vector(pct)), 1)
)
print(tabla)
cat(sprintf("\nVerificación: suma(fi) = %d  (n = %d)\n", sum(tabla$fi), length(datos)))

💡 Equivalencias Python ↔ R
value_counts() ↔ table()  ·  value_counts(normalize=True) ↔ prop.table(table(x))  ·  cumsum() ↔ cumsum() (idéntico en ambos lenguajes)  ·  crosstab() ↔ table(x, y)

⚠️ 5.6 Errores Comunes al Construir Tablas de Frecuencia

  • ❌
    Que Σfᵢ no sea igual a n
    ✅ Si las frecuencias absolutas no suman el total de datos, alguna categoría quedó mal contada o duplicada. Verifica esta suma siempre primero.
  • ❌
    Confundir frecuencia relativa con porcentaje
    ✅ hᵢ=0.40 y %ᵢ=40% son la misma proporción expresada de dos formas. Reportar "0.40%" en vez de "40%" cambia completamente el significado.
  • ❌
    Acumular frecuencias en variables nominales sin orden lógico
    ✅ "Hasta Bicicleta llevamos el 92.5%" no aporta si el orden de las categorías es arbitrario. La acumulación es natural en datos ordinales o cuantitativos.
  • ❌
    Redondear demasiado pronto
    ✅ Redondear cada hᵢ antes de sumarlas puede hacer que Σhᵢ ≠ 1 (ej. 0.40+0.23+0.20+0.10+0.08 = 1.01). Calcula con precisión completa y redondea solo al presentar la tabla final.

🎓 5.7 Puntos Clave y Quiz

Puntos Clave Semana 5

1
Una distribución de frecuencias resume un conjunto de datos mostrando cuántas veces aparece cada valor o categoría.
2
fᵢ = conteo · hᵢ = fᵢ/n = proporción · %ᵢ = hᵢ×100 · Fᵢ = suma acumulada de fᵢ.
3
Dos verificaciones obligatorias en toda tabla: Σfᵢ = n y Σhᵢ = 1 (o 100%).
4
En Python: value_counts(), cumsum(), crosstab(). En R: table(), prop.table(), cumsum().
5
La acumulación (Fᵢ, Hᵢ) solo se interpreta de forma natural cuando existe un orden lógico entre las categorías.

2. Si Σfᵢ = 38 mientras n = 40, ¿qué significa esto?
3. ¿Qué función de pandas calcula directamente la frecuencia absoluta de cada categoría?
4. ¿En qué tipo de variable tiene más sentido interpretar la frecuencia acumulada (Fᵢ)?
