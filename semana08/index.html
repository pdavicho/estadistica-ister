Histogramas y Polígonos de Frecuencia
Tema 2 · Visualizar la distribución de frecuencias agrupadas · Cierre del Tema 2
👨‍🏫 Ing. David Minango, PhD. 📅 8 de junio, 2026 ⏱ ~90 min clase · 45 min taller
📽 Presentación · 18 diapositivas · ~20 min · usa ← → o teclado ⛶ Pantalla completa

📌 8.1 ¿Qué es un Histograma?

📖 Idea central
Un histograma es la representación gráfica de una tabla de frecuencias agrupadas: sobre cada intervalo se levanta una barra cuya altura es la frecuencia (fᵢ). Las barras van pegadas porque la variable es continua.

En la Semana 7 construimos la tabla agrupada de los tiempos de respuesta de una API (40 peticiones). Hoy convertimos esa tabla en una imagen que revela de un vistazo dónde se concentran los datos y qué forma tienen. Seguimos con el mismo ejemplo para ver el puente tabla → gráfico.

📊
Histograma
Variable cuantitativa continua en intervalos. Barras pegadas; el eje X es una escala numérica.
📶
Gráfico de barras
Variable cualitativa (categorías). Barras separadas; el orden puede cambiarse.
⚠️ No los confundas
Un gráfico de "colores favoritos" no es un histograma: los colores son categorías, no una escala numérica. El histograma exige una variable numérica continua dividida en clases.

Clase (ms)xₘfᵢFᵢ

① Histograma — una barra por clase (altura = fᵢ)

Las clases 23–31 ms son las más altas (9 cada una): ahí se concentra la mayoría de las peticiones.

② Polígono de frecuencias — une los puntos (xₘ, fᵢ)

📖 Definición
El polígono de frecuencias une con rectas los puntos (marca de clase, frecuencia) y se cierra apoyándose en el eje (frecuencia 0) una clase antes de la primera y una después de la última. Resalta la tendencia y permite comparar dos distribuciones superpuestas.
Los puntos rojos están sobre la marca de clase (xₘ), no sobre los bordes del intervalo.

③ Ojiva — une los puntos (Lₛ, Fᵢ)

📖 Definición
La ojiva (polígono de frecuencias acumuladas) une los puntos (límite superior, Fᵢ). Siempre es creciente y termina en n. Responde de un vistazo: "¿cuántos datos hay por debajo de X?".
En x = 31 ms la ojiva llega a Fᵢ = 24: el 60% de las peticiones responde en menos de 31 ms.

📐 8.3 La Forma de la Distribución

Lo más valioso del histograma es que revela la forma de los datos de un solo vistazo. Estos son los patrones más frecuentes:

FormaCómo se veRelación media/medianaEjemplo típico
SimétricaLas dos mitades son espejomedia ≈ mediana ≈ modaEstaturas, errores de medición
Sesgo derecho (+)Cola larga hacia valores altosmedia > medianaIngresos, tiempos de espera
Sesgo izquierdo (−)Cola larga hacia valores bajosmedia < medianaEdad de jubilación
BimodalDos picos separados—Dos grupos mezclados
💡 Regla rápida del sesgo
La cola (el lado largo y bajo del histograma) apunta hacia donde está el sesgo. Cola a la derecha → sesgo a la derecha (positivo).
📋 Nuestro ejemplo
El histograma de los tiempos de respuesta sube rápido hasta el pico en 23–31 ms y luego baja gradualmente hacia los tiempos altos. La cola más larga está a la derecha → distribución ligeramente sesgada a la derecha: la mayoría responde rápido pero unas pocas peticiones tardan bastante más. Cuando hay sesgo, la media se desplaza hacia la cola — por eso la próxima semana estudiaremos también la mediana.

🐍 8.4 Python: Histograma con matplotlib

plt.hist() dibuja el histograma directamente desde los datos crudos usando los mismos bordes de la Semana 7. Encima superponemos el polígono uniendo las marcas de clase.

import matplotlib.pyplot as plt

datos = [20,20,20,20,21,22,23,23,23,24,24,25,26,26,26,27,27,27,28,28,
         28,29,29,30,31,32,32,34,34,34,34,36,36,36,36,39,39,39,41,41]
bordes = [19,23,27,31,35,39,43]          # mismos intervalos de la S7

# Histograma  (right=False -> [Li, Ls), igual que pd.cut)
fi, _, _ = plt.hist(datos, bins=bordes, color="#3b82f6",
                    edgecolor="white", rwidth=1.0)

# Polígono de frecuencias: marcas de clase (xm) vs fi, con extremos en 0
xm = [(bordes[i] + bordes[i+1]) / 2 for i in range(len(bordes)-1)]
plt.plot([xm[0]-4] + xm + [xm[-1]+4], [0] + list(fi) + [0],
         marker="o", color="#ef4444", label="Polígono")

plt.xlabel("Tiempo de respuesta (ms)")
plt.ylabel("Frecuencia (fi)")
plt.title("Histograma de tiempos de respuesta")
plt.legend()
plt.show()
Output
fi por clase → [6. 9. 9. 7. 4. 5.] (Σ = 40)
Se abre la figura: histograma azul (barras pegadas) con el polígono rojo encima.
💡 Histograma de frecuencia relativa
Pasa density=True a plt.hist() para que el eje Y muestre densidad (área total = 1) en lugar de frecuencias absolutas. Útil para comparar muestras de distinto tamaño.

📊 8.5 R: hist() base y ggplot2

datos <- c(20,20,20,20,21,22,23,23,23,24,24,25,26,26,26,27,27,27,28,28,
           28,29,29,30,31,32,32,34,34,34,34,36,36,36,36,39,39,39,41,41)
bordes <- seq(19, 43, by = 4)            # 19 23 27 31 35 39 43

h <- hist(datos, breaks = bordes, right = FALSE,
          col = "#3b82f6", border = "white",
          main = "Histograma de tiempos de respuesta",
          xlab = "Tiempo (ms)", ylab = "Frecuencia")

# Polígono de frecuencias superpuesto (marca de clase vs conteo)
lines(h$mids, h$counts, type = "o", col = "#ef4444", lwd = 2)

print(h$counts)   # 6 9 9 7 4 5
print(h$mids)     # 21 25 29 33 37 41
library(ggplot2)

datos <- c(20,20,20,20,21,22,23,23,23,24,24,25,26,26,26,27,27,27,28,28,
           28,29,29,30,31,32,32,34,34,34,34,36,36,36,36,39,39,39,41,41)
bordes <- seq(19, 43, by = 4)

ggplot(data.frame(t = datos), aes(x = t)) +
  geom_histogram(breaks = bordes, fill = "#3b82f6", color = "white") +
  labs(title = "Histograma de tiempos de respuesta",
       x = "Tiempo (ms)", y = "Frecuencia") +
  theme_minimal()
Output
h$counts → 6 9 9 7 4 5  ·  h$mids → 21 25 29 33 37 41
R base dibuja el histograma y lines() superpone el polígono rojo; ggplot produce la versión estilizada.
💡 Equivalencias Python ↔ R
plt.hist(x, bins) ↔ hist(x, breaks)  ·  density=True ↔ freq=FALSE  ·  el polígono se superpone con plt.plot / lines() sobre las marcas de clase.

⚠️ 8.6 Errores Comunes al Graficar

  • ❌
    Dejar espacio entre las barras
    ✅ En un histograma las barras van pegadas; el hueco es propio del gráfico de barras (datos categóricos).
  • ❌
    Clases de distinto ancho con altura = fᵢ
    ✅ Si los intervalos tienen anchos distintos, la altura debe ser la densidad (fᵢ/ancho); de lo contrario el área engaña.
  • ❌
    Trazar el polígono sobre los límites
    ✅ El polígono usa la marca de clase xₘ (punto medio), no los bordes del intervalo.
  • ❌
    Elegir mal el número de clases
    ✅ Demasiadas barras → "ruido"; muy pocas → se pierde la forma. Usa la regla de Sturges (5–12 clases) de la Semana 7.

🎓 8.7 Puntos Clave y Quiz

Puntos Clave Semana 8

1
El histograma grafica una tabla agrupada: una barra por clase, altura = fᵢ, barras pegadas (variable continua).
2
El polígono de frecuencias une los puntos (xₘ, fᵢ) y cierra en 0; resalta la tendencia y permite comparar distribuciones.
3
La ojiva une (Lₛ, Fᵢ): siempre creciente, responde "¿cuántos datos hay por debajo de X?".
4
La forma puede ser simétrica, sesgada (la cola marca el lado) o bimodal. Nuestro ejemplo está ligeramente sesgado a la derecha.
5
En código: plt.hist() (Python) y hist() / geom_histogram() (R) dibujan el histograma en una línea.

Código: genera el histograma y el polígono

Python + R · 20 min 
import matplotlib.pyplot as plt

# Calificaciones de 50 estudiantes — dataset ya dado, no lo modifiques
notas = [6,7,7,8,8,9,9,9,9,10,10,10,10,10,11,11,11,11,11,11,
         12,12,12,12,12,12,12,13,13,13,13,13,13,13,14,14,14,14,14,
         15,15,15,15,16,16,16,17,17,18,19]
bordes = [6,8,10,12,14,16,18,20]

# ── Histograma ─────────────────────────────────────────────
# Pista: el argumento que recibe los límites de las clases se llama bins
fi, _, _ = plt.hist(notas, _____=bordes, color="#3b82f6", edgecolor="white")

# ── Polígono de frecuencias ────────────────────────────────
# Pista: la marca de clase es el promedio de los dos bordes -> (a+b)/2
xm = [(bordes[i] + bordes[i+1]) / _____ for i in range(len(bordes)-1)]
# Pista: el polígono cierra en frecuencia 0 antes y después
plt.plot([xm[0]-2] + xm + [xm[-1]+2], [_____] + list(fi) + [0],
         marker="o", color="#ef4444")

plt.xlabel("Calificación"); plt.ylabel("Frecuencia")
plt.title("Histograma de calificaciones")
plt.show()
print("fi por clase:", fi, " | suma =", fi.sum())
# Calificaciones de 50 estudiantes — dataset ya dado, no lo modifiques
notas <- c(6,7,7,8,8,9,9,9,9,10,10,10,10,10,11,11,11,11,11,11,
           12,12,12,12,12,12,12,13,13,13,13,13,13,13,14,14,14,14,14,
           15,15,15,15,16,16,16,17,17,18,19)
bordes <- seq(6, 20, by = 2)

# ── Histograma  [Li, Ls) ───────────────────────────────────
# Pista: el argumento que recibe los límites se llama breaks
h <- hist(notas, _____ = bordes, right = FALSE,
          col = "#3b82f6", border = "white",
          main = "Histograma de calificaciones",
          xlab = "Calificación", ylab = "Frecuencia")

# ── Polígono: marca de clase (h$mids) vs conteo (h$counts) ──
# Pista: para unir puntos con líneas y marcadores usa type = "o"
lines(h$mids, h$counts, type = "_____", col = "#ef4444", lwd = 2)

print(h$counts)   # frecuencias por clase
cat("suma =", sum(h$counts), "\n")
