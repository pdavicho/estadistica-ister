📌 6.1 ¿Cómo se Clasifican las Distribuciones de Frecuencias?

📖 Idea central
En la Semana 5 vimos los tipos de frecuencia (las columnas: fᵢ, hᵢ, %ᵢ, Fᵢ). Esta semana vemos los tipos de distribución: las distintas formas de estructurar la tabla. No todas las distribuciones se construyen igual.

La estructura correcta de una tabla de frecuencias depende de dos decisiones:

①
Según el tipo de variable
Cualitativa → por categorías · Cuantitativa → por valores o por intervalos.
②
Según el agrupamiento
No agrupada (una fila por valor) · Agrupada (una fila por intervalo).
💡 Una mirada transversal: la acumulación
Cualquier distribución, sea del tipo que sea, puede mostrarse de forma simple (fᵢ, hᵢ) o acumulada (Fᵢ, Hᵢ).

📌 6.2 Según el Tipo de Variable

Recordando la Semana 2: la naturaleza de la variable decide cómo se organiza la tabla.

a) Variable cualitativa → por categorías

Una fila por cada categoría. Ejemplo: medio de transporte de 40 estudiantes.

b) Variable cuantitativa discreta (pocos valores) → por valores

Una fila por cada valor numérico. Ejemplo: nº de dispositivos por estudiante (n = 50). Aquí la acumulada SÍ se interpreta.

Nº dispositivosfᵢhᵢFᵢHᵢ %

Total501.00——

c) Variable cuantitativa continua → por intervalos

Muchos valores distintos: se agrupa en clases. Cada fila es un intervalo con su marca de clase (xₘ). Cómo dimensionar esos intervalos es justo el tema de la Semana 7.

⚠️ La acumulada y el tipo de variable
La frecuencia acumulada (Fᵢ) solo se interpreta de forma natural cuando existe un orden: en variables cuantitativas y cualitativas ordinales. En nominales (colores, marcas) el orden es arbitrario.

📌 6.3 No Agrupadas vs. Agrupadas

📋 No agrupada
Una fila por valor o categoría. No se pierde detalle. Ideal para cualitativas o discretas con pocos valores.
📊 Agrupada
Una fila por intervalo. Gana legibilidad y muestra la forma, a costa del detalle. Ideal para continuas o muchos valores.

Regla práctica: ¿cuándo agrupar?

SituaciónTipo de distribución
Variable cualitativa (nominal u ordinal)No agrupada (por categorías)
Cuantitativa discreta, pocos valores (≤ ~15)No agrupada (por valores)
Cuantitativa discreta con muchos valoresAgrupada (intervalos)
Cuantitativa continuaAgrupada (intervalos)
💡 Criterio guía
Si al listar los valores únicos la tabla supera ~15 filas y se vuelve difícil de leer, agrupa. El objetivo siempre es resumir sin ocultar el patrón.

📌 6.4 Distribuciones de Frecuencias Acumuladas

Cualquier distribución (agrupada o no) puede expresarse en forma acumulada: cada fila suma su frecuencia más todas las anteriores. Responde la pregunta "¿cuántos datos hay hasta este punto?".

📖 Fórmulas
Fᵢ = f₁ + f₂ + … + fᵢ (acumulada absoluta)  ·  Hᵢ = Fᵢ / n (acumulada relativa). La última Fᵢ siempre es igual a n, y la última Hᵢ es 100%.
🔗 Hacia la Semana 8
La distribución acumulada es la base de la ojiva (gráfico de frecuencias acumuladas) que veremos junto a los histogramas en la Semana 8.

💻 6.5 Código: Elegir la Tabla Correcta según el Tipo

La regla es simple: value_counts() / table() para distribuciones no agrupadas, y pd.cut() / cut() para agrupadas.

import pandas as pd

# --- NO AGRUPADA: cualitativa / discreta con pocos valores ---
transporte = pd.Series(['Bus']*16 + ['Caminando']*9 + ['Auto']*8 +
                        ['Bici']*4 + ['Taxi']*3)
print(transporte.value_counts())             # una fila por categoría

# --- AGRUPADA: continua con muchos valores ---
tiempos = pd.Series([20,21,23,24,26,27,28,29,31,34,36,39,41])   # ms
print(pd.cut(tiempos, bins=4).value_counts().sort_index())  # una fila por intervalo
# --- NO AGRUPADA: cualitativa / discreta con pocos valores ---
transporte <- c(rep("Bus",16), rep("Caminando",9), rep("Auto",8),
                rep("Bici",4), rep("Taxi",3))
table(transporte)                       # una fila por categoría

# --- AGRUPADA: continua con muchos valores ---
tiempos <- c(20,21,23,24,26,27,28,29,31,34,36,39,41)
table(cut(tiempos, breaks = 4))         # una fila por intervalo

💡 Equivalencias Python ↔ R
value_counts() ↔ table()  ·  pd.cut() ↔ cut(). La elección de la función refleja directamente el tipo de distribución que quieres construir.

🎓 6.6 Puntos Clave y Quiz

Puntos Clave Semana 6

1
La estructura de la tabla depende del tipo de variable y de si conviene agrupar.
2
Cualitativa → por categorías; discreta con pocos valores → por valores; continua o muchos valores → por intervalos.
3
Distribución no agrupada (fila por valor) vs agrupada (fila por intervalo).
4
Toda distribución puede ser simple o acumulada (Fᵢ, Hᵢ); la acumulada exige un orden.
5
En código: value_counts()/table() = no agrupada · pd.cut()/cut() = agrupada.

Caso: Encuesta de satisfacción en un tecnológico

APP · Desarrollo · probabilístico + no probabilístico
Escenario: Un instituto tiene N = 1500 estudiantes distribuidos en 4 carreras: Sistemas = 600, Industrial = 450, Mecánica = 300 y Ambiental = 150. Se desea aplicar una encuesta de satisfacción a una muestra de n = 150 estudiantes. Diseña el muestreo y responde.
  1. 1
    Calcula la fracción de muestreo f = n/N. ¿Qué porcentaje de la población vas a encuestar?
    f = n/N. Multiplica por 100 para el porcentaje.
  2. 2
    Aplica muestreo estratificado proporcional: ¿cuántos estudiantes encuestas de cada carrera? Muestra el cálculo y verifica que sumen 150.
    nₕ = Nₕ × (n/N). Por ejemplo, Sistemas = 600 × f.
  3. 3
    Si en lugar de estratificar usaras muestreo sistemático sobre la lista completa, ¿cuál es el intervalo de selección k? ¿Cómo eliges el punto de arranque?
    k = N/n. El arranque es un número aleatorio entre 1 y k; luego seleccionas cada k-ésimo elemento.
  4. 4
    El coordinador propone "encuestar solo a los estudiantes que estén en la cafetería al mediodía". Identifica qué técnica es y qué sesgo introduce.
    ¿Es probabilística o no? Piensa en qué estudiantes quedan sistemáticamente excluidos.
  5. 5
    Justifica: ¿por qué el muestreo estratificado es preferible al aleatorio simple en este caso? ¿Qué garantiza respecto a las carreras pequeñas como Ambiental?

Código: simula el muestreo con Python / R

Python + R · completa los blancos 
import numpy as np
rng = np.random.default_rng(42)

# Datos del caso — no los modifiques
N = 1500
n = 150
estratos = {'Sistemas': 600, 'Industrial': 450, 'Mecanica': 300, 'Ambiental': 150}

# ── Fracción de muestreo ──────────────────────────────────
# Pista: la fracción es n dividido para N
f = n / _____
print(f"Fracción de muestreo = {f:.2%}")

# ── Muestreo estratificado proporcional ───────────────────
print("\nReparto estratificado proporcional:")
total = 0
for carrera, Nh in estratos.items():
    # Pista: nº por estrato = tamaño del estrato * fracción
    nh = round(Nh * _____)
    total += nh
    print(f"  {carrera:12s}: {nh}")
print(f"  Suma = {total}  (debe ser {n})")

# ── Muestreo sistemático ──────────────────────────────────
# Pista: el intervalo k es N dividido para n
k = N // _____
arranque = rng.integers(1, k + 1)        # punto de partida aleatorio
seleccion = list(range(arranque, N + 1, k))
print(f"\nSistemático: k = {k}, arranque = {arranque}, seleccionados = {len(seleccion)}")
set.seed(42)

nh <- round(estratos * _____)

# ── Muestreo sistemático ──────────────────────────────────
# Pista: el intervalo k es N dividido para n
k <- N %/% _____
arranque <- sample(1:k, 1)
seleccion <- seq(arranque, N, by = k)
cat("Sistemático: k =", k, " arranque =", arranque,
    " seleccionados =", length(seleccion), "\n")
