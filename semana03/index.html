📌 3.1 ¿Por qué visualizar datos?

📖 Principio fundamental
Una tabla con 10,000 filas es imposible de interpretar de un vistazo. El gráfico correcto revela patrones, tendencias y anomalías en segundos.

La representación gráfica es la última etapa de la estadística descriptiva. Permite comunicar hallazgos a audiencias técnicas y no técnicas de forma efectiva.

🔍
Descubrir patrones
Tendencias, ciclos, outliers que los números ocultan.
📣
Comunicar resultados
Presentar conclusiones a directivos, clientes, equipos.
🚨
Detectar anomalías
Errores, fraudes o fallos visibles inmediatamente.

📌 3.2 Tipos de Gráficos Estadísticos

GráficoTipo de variable¿Para qué sirve?Ejemplo en sistemas
📊 BarrasCualitativaComparar categorías entre síN° tickets por tipo (Bug/Feature/Soporte)
📈 LíneasCuantitativaMostrar evolución en el tiempoLatencia del servidor por hora
🔵 HistogramaCuantitativaDistribución de una variable continuaDistribución de tiempos de respuesta (ms)
🥧 Pastel/TortaCualitativaComposición porcentual (máx. 5 categorías)% uptime por servicio
📦 BoxplotCuantitativaDistribución + outliers + cuartilesTiempo de carga por país
⚫ DispersiónAmbasRelación entre dos variablesCPU% vs tiempo de respuesta
⚠️ Error común
Los gráficos de pastel con más de 5 categorías son ilegibles. Prefiere un gráfico de barras horizontales. El histograma es para datos continuos, no para categorías.

📌 3.3 ¿Qué gráfico usar?

La elección del gráfico correcto depende del tipo de variable y la pregunta que quieres responder:

¿Comparar categorías?
→ Gráfico de barras
Ej: Número de errores por módulo del sistema.
¿Ver evolución temporal?
→ Gráfico de líneas
Ej: Tráfico de red por hora a lo largo de una semana.
¿Distribución de valores?
→ Histograma o Boxplot
Ej: Distribución de tiempos de respuesta del API.
¿Relación entre variables?
→ Diagrama de dispersión
Ej: ¿A mayor uso de CPU, mayor latencia?

🐍 3.4 Python: matplotlib y seaborn

Python ofrece dos librerías complementarias: matplotlib para control total y seaborn para gráficos estadísticos elegantes con menos código.

import matplotlib.pyplot as plt
import numpy as np
np.random.seed(42)

fig, axes = plt.subplots(2, 3, figsize=(15, 9))
fig.suptitle('Monitoreo de Infraestructura Tech', fontsize=14, fontweight='bold')

# 1. BARRAS: Tickets por tipo
tipos = ['Bug', 'Feature', 'Soporte', 'Seguridad']
tickets = [145, 98, 234, 45]
axes[0,0].bar(tipos, tickets, color=['#ef4444','#3b82f6','#22c55e','#f59e0b'])
axes[0,0].set_title('Tickets por Tipo')
axes[0,0].set_ylabel('Cantidad')

# 2. LÍNEAS: Latencia por hora
horas = np.arange(0, 24)
latencia = 50 + 30*np.sin(horas*np.pi/12) + np.random.normal(0, 5, 24)
axes[0,1].plot(horas, latencia, 'o-', color='#2563eb', linewidth=2, markersize=4)
axes[0,1].axhline(y=100, color='red', linestyle='--', alpha=0.7, label='SLA máx')
axes[0,1].set_title('Latencia por Hora (ms)')
axes[0,1].legend()

# 3. HISTOGRAMA: Tiempos de respuesta
tiempos = np.random.gamma(2, 50, 2000)
axes[0,2].hist(tiempos, bins=40, color='#3b82f6', alpha=0.8, edgecolor='white')
axes[0,2].set_title('Distribución Tiempos de Respuesta')
axes[0,2].set_xlabel('ms')

# 4. BOXPLOT: CPU por servidor
servidores = [f'SRV-{i}' for i in range(1,6)]
cpu_data = [np.random.normal(m, 8, 100) for m in [45,62,38,78,55]]
axes[1,0].boxplot(cpu_data, labels=servidores)
axes[1,0].set_title('Uso CPU por Servidor (%)')

# 5. PASTEL: Causas de downtime
causas = ['Hardware', 'Software', 'Red', 'Humano']
pct = [35, 40, 15, 10]
axes[1,1].pie(pct, labels=causas, autopct='%1.0f%%', startangle=90,
              colors=['#3b82f6','#22c55e','#f59e0b','#ef4444'])
axes[1,1].set_title('Causas de Downtime')

# 6. DISPERSIÓN: CPU vs Latencia
cpu = np.random.normal(60, 15, 200)
lat = cpu * 2.5 + np.random.normal(0, 30, 200)
axes[1,2].scatter(cpu, lat, alpha=0.5, color='#8b5cf6')
axes[1,2].set_title('CPU (%) vs Latencia (ms)')
axes[1,2].set_xlabel('CPU %'); axes[1,2].set_ylabel('Latencia ms')

plt.tight_layout()
plt.savefig('dashboard_infra.png', dpi=150, bbox_inches='tight')
plt.show()
print("✅ Dashboard guardado como 'dashboard_infra.png'")
import seaborn as sns
import pandas as pd
import matplotlib.pyplot as plt
import numpy as np
np.random.seed(42)

# Dataset de monitoreo de servidores
n = 500
df = pd.DataFrame({
    'servidor': np.random.choice(['Web-1','Web-2','DB-1','Cache-1'], n),
    'cpu_pct': np.random.beta(2, 3, n) * 100,
    'ram_pct': np.random.beta(3, 2, n) * 100,
    'latencia_ms': np.abs(np.random.lognormal(4, 0.5, n)),
    'estado': np.random.choice(['OK','Warning','Critical'], n, p=[.75,.18,.07])
})

sns.set_theme(style='whitegrid', palette='deep')
fig, axes = plt.subplots(1, 3, figsize=(15, 5))

# Boxplot de latencia por servidor
sns.boxplot(data=df, x='servidor', y='latencia_ms', ax=axes[0], palette='Blues')
axes[0].set_title('Latencia por Servidor (Boxplot)')

# Histograma con curva de densidad
sns.histplot(data=df, x='cpu_pct', hue='estado', bins=25, ax=axes[1], kde=True)
axes[1].set_title('Distribución CPU% por Estado')

# Dispersión CPU vs RAM
sns.scatterplot(data=df, x='cpu_pct', y='ram_pct', hue='estado',
                palette={'OK':'green','Warning':'orange','Critical':'red'},
                alpha=0.6, ax=axes[2])
axes[2].set_title('CPU% vs RAM% por Estado')

plt.tight_layout()
plt.show()

📊 3.5 R: ggplot2

ggplot2 usa la "Gramática de Gráficos": datos + estética (aes) + geometría (geom) + opciones. Es el estándar en R para visualización estadística.

library(ggplot2)
library(dplyr)
set.seed(42)

# Dataset de monitoreo
n <- 500
df <- data.frame(
  servidor = sample(c("Web-1","Web-2","DB-1","Cache-1"), n, replace=TRUE),
  cpu_pct  = rbeta(n, 2, 3) * 100,
  latencia = abs(rlnorm(n, 4, 0.5)),
  estado   = sample(c("OK","Warning","Critical"), n, replace=TRUE, prob=c(.75,.18,.07))
)

# 1. HISTOGRAMA con densidad
p1 <- ggplot(df, aes(x = latencia, fill = estado)) +
  geom_histogram(bins = 30, alpha = 0.7, position = "identity") +
  scale_fill_manual(values = c(OK="seagreen", Warning="orange", Critical="red")) +
  labs(title = "Distribución de Latencia por Estado del Servidor",
       x = "Latencia (ms)", y = "Frecuencia") +
  theme_minimal(base_size = 12)

# 2. BOXPLOT por servidor
p2 <- ggplot(df, aes(x = servidor, y = cpu_pct, fill = servidor)) +
  geom_boxplot(alpha = 0.8, outlier.color = "red") +
  labs(title = "Distribución de CPU% por Servidor",
       x = "Servidor", y = "CPU (%)") +
  theme_minimal() + theme(legend.position = "none")

# 3. DISPERSIÓN CPU vs Latencia
p3 <- ggplot(df, aes(x = cpu_pct, y = latencia, color = estado)) +
  geom_point(alpha = 0.5, size = 1.5) +
  geom_smooth(method = "lm", se = FALSE, color = "black", linetype = "dashed") +
  scale_color_manual(values = c(OK="seagreen", Warning="orange", Critical="red")) +
  labs(title = "Correlación: CPU% vs Latencia (ms)", x = "CPU (%)", y = "Latencia (ms)") +
  theme_minimal()

print(p1); print(p2); print(p3)

Caso: Dashboard de Monitoreo de Red — TechEcuador S.A.

ABP · 20 min
Escenario: TechEcuador S.A. tiene 8 servidores web. El equipo de operaciones necesita un dashboard visual para el reporte semanal del gerente de TI. Tienes acceso a logs de la última semana: latencia, uso de CPU, tipos de errores y disponibilidad por servidor.
  1. 1
    Identifica el tipo de variable para cada métrica: latencia_ms, servidor_id, estado (OK/Warning/Critical), cpu_pct, errores_por_hora.
    latencia_ms: Cuantitativa Continua (Razón) · servidor_id: Cualitativa Nominal · estado: Cualitativa Ordinal · cpu_pct: Cuantitativa Continua (Razón) · errores_por_hora: Cuantitativa Discreta (Razón)
  2. 2
    Para cada variable, ¿qué gráfico elegirías para el dashboard del gerente?
    latencia_ms → Histograma + Boxplot (distribución y outliers) · servidor_id/estado → Barras (comparación) · cpu_pct a lo largo del tiempo → Líneas · errores por tipo → Barras horizontales
  3. 3
    El gerente dice: "Quiero ver en un solo gráfico si todos los servidores tienen latencia similar." ¿Qué gráfico usas?
    Boxplot múltiple: un box por servidor en el eje X, latencia_ms en el eje Y. Permite comparar distribuciones y ver outliers por servidor simultáneamente.
  4. 4
    ¿Qué conclusión puedes sacar si en el gráfico de dispersión (CPU% vs latencia_ms) los puntos forman una línea diagonal hacia arriba?
    Existe una correlación positiva: a mayor uso de CPU, mayor latencia. Esto sugiere que el problema de rendimiento puede ser cuello de botella en procesamiento.
  5. 5
    El 70% de los errores son de tipo "Timeout". ¿Usarías un gráfico de pastel o de barras? Justifica.
    Barras: aunque haya pocas categorías, las barras permiten leer valores exactos más fácilmente. El pastel es adecuado solo cuando las proporciones son el mensaje principal y hay ≤5 categorías.
